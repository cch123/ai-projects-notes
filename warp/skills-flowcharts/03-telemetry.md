# 三、遥测与产品信号

用于设计和接入 telemetry。重点不是把事件发出去，而是确认事件语义、payload 最小化、UGC 标记和启用范围。


### `add-telemetry`

用途：为用户行为或系统事件增加遥测事件。

```text
+------------------------------------------------------------+
| 触发：需要追踪新功能使用情况、错误路径、性能或系统状态。     |
+-----------------------------+------------------------------+
                              |
                              v
+------------------------------------------------------------+
| 0. 先设计遥测，不先写代码                                   |
| 必须和用户/需求方确认：                                     |
| - 要追踪哪些事件？                                          |
| - 每个事件何时触发？                                        |
| - payload 需要哪些字段？                                    |
| - 字段是否包含 UGC、路径、命令、账号信息或其他敏感信息？     |
| - 这些数据会用于什么判断或指标？                            |
+-----------------------------+------------------------------+
                              |
                              v
+------------------------------------------------------------+
| 判断：使用已有 telemetry module 还是新建？                  |
|                                                            |
| 使用已有 module，如果：                                     |
| - 事件属于已有功能域。                                      |
| - 文件中已有同类 FeatureTelemetryEvent enum。                |
| - 新事件和已有事件会由同一 owner 维护。                     |
|                                                            |
| 新建 telemetry.rs，如果：                                   |
| - 功能域独立。                                              |
| - 塞进现有 module 会扩大无关 enum。                          |
| - 新 feature 需要独立注册和描述。                           |
+-----------------------------+------------------------------+
                              |
                              v
+------------------------------------------------------------+
| 1. 定义 enum                                                |
| app/src/<feature>/telemetry.rs                              |
| - enum 名称按功能域命名，例如 YourFeatureTelemetryEvent      |
| - variant 按用户动作/系统结果命名，例如 ActionStarted        |
| - payload 字段只保留分析必需信息。                          |
+-----------------------------+------------------------------+
                              |
                              v
+------------------------------------------------------------+
| 2. 实现 TelemetryEvent                                      |
| 必填方法：                                                  |
| - name()                                                    |
| - payload()                                                 |
| - description()                                             |
| - enablement_state()                                        |
| - contains_ugc()                                            |
| - event_descs()                                             |
+-----------------------------+------------------------------+
                              |
                              v
+------------------------------------------------------------+
| 判断：enablement_state 选什么？                             |
|                                                            |
| EnablementState::Always，如果：                             |
| - 事件对所有 channel 都有意义。                              |
| - 不依赖未发布 feature。                                    |
|                                                            |
| EnablementState::Flag(FeatureFlag::X)，如果：                |
| - 事件只和 gated feature 有关。                              |
| - feature 关闭时发送事件会产生误导。                        |
|                                                            |
| EnablementState::Channel(Channel::Dev)，如果：               |
| - 只用于开发调试或内部诊断。                                |
+-----------------------------+------------------------------+
                              |
                              v
+------------------------------------------------------------+
| 判断：contains_ugc() 是否为 true？                          |
|                                                            |
| true，如果 payload 含：                                     |
| - 用户输入的命令、prompt、文件内容、路径、URL、文本片段。    |
| - 任何可能反映用户 workspace 或私有数据的字段。              |
|                                                            |
| false，如果 payload 只含：                                  |
| - 枚举状态、布尔值、耗时、计数、错误类别等非用户内容。       |
+-----------------------------+------------------------------+
                              |
                              v
+------------------------------------------------------------+
| 3. 实现 TelemetryEventDesc for discriminants                |
| - name: Feature.Action.Result                               |
| - description: 解释事件语义，不写实现细节                   |
| - enablement_state: 与事件暴露范围一致                      |
+-----------------------------+------------------------------+
                              |
                              v
+------------------------------------------------------------+
| 4. 注册事件                                                 |
| warp_core::register_telemetry_event!(YourFeatureTelemetryEvent) |
+-----------------------------+------------------------------+
                              |
                              v
+------------------------------------------------------------+
| 判断：从哪里发送事件？                                      |
|                                                            |
| send_telemetry_from_ctx!，如果：                            |
| - 当前代码有 ViewContext 或 ModelContext。                   |
|                                                            |
| send_telemetry_from_app_ctx!，如果：                         |
| - 当前代码只有 AppContext。                                 |
+-----------------------------+------------------------------+
                              |
                              v
+------------------------------------------------------------+
| 验证                                                        |
| - cargo run --features log_named_telemetry_events            |
| - 触发行为，确认事件名和 payload 出现在日志中。              |
| - 检查 contains_ugc、enablement_state 和 payload 最小化。    |
+------------------------------------------------------------+
```

解释：原 skill：[add-telemetry](https://github.com/warpdotdev/warp/blob/master/.agents/skills/add-telemetry/SKILL.md)。

要点：
- 适用场景：需要衡量新功能使用率、关键路径转化、错误发生位置、性能或系统状态，并且这些数据会被实际用于产品或工程判断。
- 核心判断：先定义事件语义，再写代码。每个事件都应回答“为什么要发、何时发、payload 为什么需要这些字段、是否包含 UGC”。
- 关键产物：feature-specific telemetry enum、`TelemetryEvent`/`TelemetryEventDesc` 实现、`register_telemetry_event!` 注册，以及业务代码中的发送点。
- 容易踩的坑：为了调试随手加事件、payload 携带过多用户内容、`contains_ugc()` 标错、或把不同功能域的事件塞进一个巨大 enum，都会降低 telemetry 的可用性和安全性。
